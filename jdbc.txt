1. Чтобы из контейнера с клоудерой обратиться к другим контейнерам с реляционными базами данных, сперва необходимо узнать их ip адреса в подсети докера следующей командой:
    docker inspect -f '{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}' *CONTAINER_ID*

2. Затем необходимо узнать на каком порту контейнера висит база, изучив вывод команды docker ps -a.
    У Oracle - 1521, MsSql - 1433.

3. Захордкодить в классе JDBC соответсвующие имя хоста и порт.

4. Сделать mvn clean install.

5. Переместить sparksca-1.0-SNAPSHOT.jar на ноду кластера.

6. В ту же директорию поместить драйвера для Oracle и MsSql из директории lib.

7. Запускать спарк джобу следующей командой:
    spark2-submit --master yarn --class com.spark.project.JDBC --driver-class-path ojdbc6.jar,mssql-jdbc-7.0.0.jre8.jar --jars ojdbc6.jar,mssql-jdbc-7.0.0.jre8.jar sparksca-1.0-SNAPSHOT.jar

Замечание: упоминание драйверов через мавен при спарк сабмите ничего не даёт, поэтому соответствующие мавен зависимости, по сути, нужны только для локальных запусков через IDE.
